FlytBase Drone Management Platform

Documentation for drone fleet management solution

Project Overview

FlytBase Drone Management Platform is a comprehensive web-based solution designed to manage drone fleets, schedule missions, track on-field operations, and provide real-time analytics. This application serves as a central hub for drone operators, fleet managers, and administrators to coordinate operations effectively.

Built with modern web technologies and following best practices, the platform offers a responsive, intuitive interface with powerful features to streamline drone operations management.

Technologies Used

Frontend Framework

  • Angular 19 (with standalone components)
  • TypeScript
  • RxJS for reactive programming

UI Components

  • PrimeNG v19 component library
  • Tailwind CSS for utility-first styling
  • Custom SCSS for theme management

Maps & Charts

  • Google Maps API integration
  • Dynamic charts with Chart.js
  • Syncfusion Scheduler for calendar

Build & Deployment

  • Angular CLI (@angular/cli)
  • Automated deployment via Netlify
  • Modular architecture with lazy loading

Getting Started

Follow these steps to set up the FlytBase Drone Management Platform on your local environment:

Prerequisites

  • Node.js (v16+) and npm (v8+)
  • Angular CLI (@angular/cli)
  • Git

Installation

1. Install Angular CLI globally (if not already installed):

npm install -g @angular/cli

2. Clone the repository:

git clone https://github.com/flytbase/flytbase-drone.git
cd flytbase-drone/front-end

3. Install dependencies:

npm install

4. Start the development server:

ng serve

5. Access the application at http://localhost:4200

Environment Configuration

For Google Maps integration and other services, you'll need to configure environment variables:

  1. Copy src/environments/environment.example.ts to src/environments/environment.ts
  2. Update the API keys and service endpoints as needed

Key Directories

  • src/app/layout: Contains all layout-related components, including main shell, navigation, sidebar, and theme settings
  • src/app/pages: Feature modules organized by functionality (dashboard, planning, management, etc.)
  • src/app/service: Shared services for data access, authentication, and utilities
  • src/assets: Static assets including images, icons, and theme configurations
Configuration

Theme Customization

The application supports dynamic theme switching and color customization:

  • Theme configurations are managed by the Layout Service in src/app/layout/service/layout.service.ts
  • PrimeNG themes are defined in src/assets/layout/styles/theme
  • Dark mode is supported with dedicated theme variables

Menu Configuration

Application menu structure is defined in src/app/layout/component/app-menu/app.menu.ts. Modify the menu structure by updating the model property.

Routing

The application uses Angular's lazy loading for efficient module loading:

  • Main routes are defined in src/app.routes.ts
  • Feature module routes are defined in their respective modules

Build for Production

Create a production build:

ng build --configuration production

The build output will be stored in the dist/ directory.

Netlify Deployment

The project is configured for automated deployment through Netlify:

  • Commits to the main branch trigger automatic builds and deployments
  • Deployment configurations are stored in the netlify.toml file
  • Environment-specific variables are managed through the Netlify dashboard
